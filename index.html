SOP & Governance

Jobdesk & Tata Kelola Data Tim Pendataan HI-FEST 2026

1. Pembagian Jobdesk & Hak Akses (Siapa Melakukan Apa?)

Untuk menjaga keamanan, pengelolaan data dibagi menjadi beberapa level akses:

▪ Azriel & Noura (Super Admin - Akses: Editor/Full Control)
  • Mengelola Master Database (File A) dan memastikan sistem Mirroring ke File B berjalan lancar.
  • Bertanggung jawab atas otomatisasi Real-time Alert (notifikasi WA) dan Backup Plan harian (setiap pukul 00.00 WIB).
  • Melakukan screening, validasi berkas, dan memegang kendali penuh atas perubahan status data.
▪ Ketua Pelaksana & Bendahara (Management - Akses: View Only Seluruh Data)
  • Memantau perkembangan jumlah pendaftar secara keseluruhan.
  • Bendahara melakukan cross-check uang masuk berdasarkan data validasi dari Tim Pendataan.
▪ Koordinator Lomba (User - Akses: View Only Terbatas)

Hanya bisa melihat data khusus cabang lombanya sendiri melalui File B (Importrange) atau Dashboard untuk kebutuhan teknis pertandingan.

2. Alur Kerja Harian (SOP Step-by-Step)
[Peserta Isi GForm] ➔ [Notifikasi WA Masuk] ➔ [Ronda Data] ➔ [Assign Status: Hijau/Follow Up] ➔ [File B]
  1. Entry & Alert: Peserta mendaftar via Google Form. Notifikasi otomatis masuk ke grup WA Tim Pendataan.
  2. Standardisasi Format: Cek apakah input data sudah seragam:
    • Nama: Setiap awal kata kapital.
    • No. Telp: Menggunakan kode negara tanpa "+" (contoh: 62 atau 20).
    • Berkas: Format PDF.
  3. Ronda Screening (Ceklis Validasi):
    • Nama: Pastikan menggunakan nama asli (bukan nama samaran/alay).
    • Bukti Bayar: Cek nominal (apakah ada kode unik seperti Rp50.006) dan pastikan struk asli (cek resolusi gambar, bukan editan).
    • Link Berkas: Pastikan link Google Drive berkas tidak dikunci dan bisa dibuka.
  4. Sistem Flagging (Pengambilan Keputusan):
    • Lolos (YES): Jika semua aman, ganti warna baris di Spreadsheet menjadi Hijau (Verified).
    • Bermasalah (NO): Jika berkas salah/dikunci, tandai dengan warna Merah, lalu Admin segera hubungi peserta via WA untuk Follow Up.
3. Protokol Keamanan & Sistem Cadangan

▪ Prinsip Satu Pintu: Data masuk hanya lewat satu GForm resmi, dan data keluar untuk divisi lain dikunci menggunakan rumus =IMPORTRANGE (File B - View Only). Ini mencegah pihak luar mengubah atau merusak data asli di File A.

▪ Komitmen NDA Internal: Azriel dan Noura berkomitmen menjaga kerahasiaan data pendaftar. Dilarang keras mengunduh (download) data ke perangkat pribadi tanpa izin.

▪ Mitigasi Risiko: Data diekspor secara otomatis/berkala setiap pukul 00.00 WIB ke cloud cadangan atau penyimpanan fisik untuk menghindari kehilangan data akibat kendala akun.

4. Timeline Kerja & Sinkronisasi (Jadwal Setor Data)

Agar koordinasi antardivisi tidak berantakan, berikut jadwal wajib setor data:

▪ Setiap Hari (23.00 WIB): Update total pendaftar harian ke grup inti panitia.

▪ 19 April: Sinkronisasi data awal dengan Divisi Bendahara untuk mencocokkan uang masuk.

▪ 05 Mei: Serah terima data final 100% (Verified & dikunci) ke Divisi Lomba untuk pembuatan nomor peserta dan bagan pertandingan.

▪ 07 Mei: Penyerahan data pemenang ke Divisi Humas untuk bahan konten pengumuman.
